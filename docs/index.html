the gap

Three instruments, none of them shows the whole sea.

Every existing tool holds one piece. Nothing joins GitHub context with agent cost — that join is the product.

github actions tab

Knows the runs, not the cost

Run status lives repo-by-repo. The action exposes no per-run token or cost outputs — clicking through 200 repos is the state of the art.

anthropic console

Knows the spend, not the repos

Org-level usage and cost, but no idea which repo, workflow, or pull request an agent session belonged to.

scuttledeck

Joins run ↔ session ↔ dollar

A correlator ties each OpenTelemetry session to its exact workflow run, PR, and invoice line — with provenance on every figure.

how it works

One webhook, one workflow step, one poller.

Install a read-only GitHub App

Webhooks stream run and PR events; a scanner parses workflow YAML to discover every repo using the action — GitHub has no API for that.

Drop in scuttledeck/setup@v1

One step before the action. It switches on Claude Code's built-in OTel export and tags every metric with the run id — tokens and cost arrive pre-joined.

Optionally add an Admin API key

Anthropic's Analytics API fills in per-key daily cost with zero workflow changes, and the cost report reconciles estimates against your actual invoice.

# .github/workflows/claude.yml
- uses: scuttledeck/setup@v1
  with:
    endpoint: https://scuttledeck.internal.dev
    token: ${{ secrets.SCUTTLEDECK_TOKEN }}

- uses: anthropics/claude-code-action@v1
  with:
    anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}

# that's it — the next run reports
# its true token cost, per PR.

the deck oath

Built to be trusted with your CI.

§1 Metadata only

No prompt or code content is ever stored. Counts, costs, durations, statuses, PR numbers — log bodies are dropped at ingest.

§2 Read-only, forever

The GitHub App asks for read scopes and nothing else. Scuttledeck never writes to your repos. That's a permanent design constraint, not a roadmap item.

§3 Your infra, your data

docker compose up and it runs on your network. Secrets encrypted at rest, ingest tokens hashed and revocable.
